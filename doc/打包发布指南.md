# 打包发布指南（保护源码版）

## 方案说明

本方案采用**清单文件驱动**，实现：
- ✅ **源码保护**：.py 源文件不打包，只打包 .pyc 字节码
- ✅ **自动发现**：运行从 test_modules.txt 读取模块列表并导入
- ✅ **双环境兼容**：开发环境用 pkgutil 扫描，打包环境用清单导入

---


## 故障排查

### 问题1：打包后 exe 报错 "No module named 'misc.battery.xxx'"

**原因：** hiddenimports 缺少顶级包或清单未包含该模块

**解决：**
1. 检查 `build.spec` 的 `hiddenimports` 是否包含该模块的顶级包（如 `misc`）
2. 检查 `test_modules.txt` 是否包含该模块的完整路径
3. 重新打包（`--clean`）

---

### 问题2：`test_modules.txt` 未打包进 _internal

**原因：** `build.spec` 的 `datas` 配置错误

**解决：** 确认 spec 中有：
```python
datas=[
    ('tools', 'tools'),
    ('test_modules.txt', '.'),  # 这一行必须存在
],
```

---

### 问题3：打包后找不到测试用例（注册 0 个）

**原因1：** `test_modules.txt` 路径错误或内容为空

**排查：**
```powershell
# 检查清单文件是否在 _internal
type dist\RV1126B_RK3576_Test_Tool\_internal\test_modules.txt

# 应该看到 22 行模块名
```

**解决：** 重新生成清单文件并清理打包

---

**原因2：** 模块名格式错误

**排查：** 清单中的模块名必须是完整导入路径（相对项目根目录的包路径）：
```
✅ 正确：misc.battery.test_battery_charge_status_test
❌ 错误：test_battery_charge_status_test
❌ 错误：misc.battery.test_battery (猜测)
```

**解决：** 使用 `verify_manifest.py` 验证（只能在开发环境运行）

---

### 问题4：.py 源码文件被打包（源码未保护）

**原因：** 之前使用了 `datas` 复制测试模块目录

**解决：**
1. 检查 `build.spec` 的 `datas` 列表：
   - ✅ 只包含 `('tools', 'tools')` 和 `('test_modules.txt', '.')`
   - ❌ 不包含 `('misc', 'misc')`、`('btwifi', 'btwifi')` 等

2. 如果错误添加了，**删除所有测试模块目录的 datas 条目**

3. 重新打包（`--clean`）

### 方法2：使用生成脚本（自动化，可选）

如果有很多新增用例，可以编写脚本自动生成清单（当前项目未提供，可按需创建）：

```python
# scripts/update_manifest.py
import os

ROOT = '.'  # 项目根目录
TEST_ROOTS = ['misc', 'btwifi', ...]

modules = []
for root_name in TEST_ROOTS:
    for dirpath, dirnames, filenames in os.walk(os.path.join(ROOT, root_name)):
        for f in filenames:
            if f.startswith('test_') and f.endswith('.py'):
                # 计算模块名...
                modules.append(module_name)

# 写入 test_modules.txt
with open('test_modules.txt', 'w') as f:
    f.write('\n'.join(sorted(set(modules))))

print(f"更新清单：{len(modules)} 个模块")
```

## 方案对比总结

| 维度 | datas复制源码 | 清单文件（当前）✅ |
|------|--------------|-------------------|
| 源码保护 | ❌ 明文暴露 | ✅ .pyc 保护 |
| 打包体积 | 较大（+ ~100KB） | 更小（.pyc 压缩） |
| 加载速度 | 慢（.py 解释） | 快（.pyc 直接加载） |
| 自动发现 | ✅ 文件扫描 | ⚠️ 需更新清单 |
| 调试便利 | ✅ 可读源码 | ❌ 需反编译 |
| 维护成本 | 低 | 中（维护清单） |
| 失败风险 | 低 | 中（清单错误） |

## WiFi 扫描功能测试

### 前置条件

1. 确保系统有无线网卡（USB 或内置）
2. **Windows 系统需要管理员权限**（右键 exe → "以管理员身份运行"）
3. 确保 `requirements.txt` 已安装 `pywifi` 和 `comtypes`

### 测试步骤

```powershell
# 1. 启动 exe（以管理员身份运行）
.\RV1126B_RK3576_Test_Tool.exe

# 2. 点击 "WiFi扫描" 按钮
# 预期：表格显示以 Xb 开头的 WiFi 热点列表

# 3. 查看表格列：
#    - 列0: WiFi名称（如 "XbotGo-0012E3"）
#    - 列1: 设备类型（Chameleon / Falcon，根据名称前缀自动推导）

# 4. 选中一个热点，点击 "连接设备"
# 预期：USB 设备匹配逻辑与蓝牙相同，连接成功
```

### 注意事项

- WiFi 扫描仅用于**发现设备热点**，实际连接仍通过 USB ADB
- 如果提示"需要管理员权限"，请重启 exe 并使用管理员身份
- WiFi 扫描结果可能受环境干扰，如未发现热点请靠近设备

## 故障排查 - WiFi 相关

### 问题1：WiFi 扫描失败：`'Profile' object has no attribute 'channel'`

**原因：** Windows 的 `pywifi` 库不支持 `channel` 属性。  
**解决：** 已修复，使用最新版 `wifi_service.py`（已移除 channel 字段）。

---

### 问题2：启动时警告 `pywifi 模块未安装` 但已安装

**原因：** `pywifi` 依赖 `comtypes` 库，可能未安装。  
**诊断：**
```powershell
python -c "import pywifi"
# 如果报错 ModuleNotFoundError: No module named 'comtypes'
```
**解决：**
```powershell
pip install comtypes
```

---

### 问题3：WiFi 扫描提示"需要管理员权限"

**原因：** Windows WiFi API 需要 elevated 权限。  
**解决：** 右键 exe → "以管理员身份运行"。

---

### 问题4：WiFi 扫描返回空列表

可能原因：
1. 无线网卡被禁用或未安装驱动
2. 周围没有 Xb 开头的 WiFi 热点
3. WiFi 适配器未开启（检查系统 WiFi 开关）
4. 权限不足（使用管理员运行）

**诊断：**
```powershell
# 手动测试 pywifi
python -c "from commons import WiFiService; print(WiFiService.scan_networks())"
```
