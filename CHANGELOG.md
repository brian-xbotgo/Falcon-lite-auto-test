# 变更日志

## [Unreleased]

### Added
- ✨ 新增5个自动化测试用例：自动追踪、记分牌&水印录制、电子围栏、1080P30fps录制、2K30fps录制
- 📊 测试用例数据模型扩展：新增 `test_case_number` 字段（字符串类型），支持初始化留空
- 📋 拓展测试报告输出能力：在现有 HTML 格式报告基础上，新增表格格式报告，保证两份文件内容完全一致，实现双格式同步生成
- 📡 设备扫描扩展：支持22个存量测试用例的 `test_case_number` 字段梳理，统一暂不填写编号待后续人工维护

### Changed
- 🔧 重构四类多媒体自动化测试：音频录制、视频录制、拍照功能、M键录像
- 🔧 摒弃原通过文件检索 + 时间校验的缺陷逻辑，统一改为测试前订阅 ACR 主题并输出至临时文本文件
- 🔧 通过解析订阅内容提取资源路径，结合对应检测工具、设备专属版本工具完成用例自动化结果判定与日志备注记录
- 🔧 适配 Falcon、Falcon-Air 两款设备
- 🔧 优化M键自动化测试执行流程，精简指令执行等待间隔，有效缩减整体测试耗时
- 🔧 规范各类用例的 MQTT 指令交互、工具推送、文件格式校验、正则匹配及结果判定规则
- 🔧 自动追踪、记分牌&水印录制测试从人工改为自动化
- 🔄 `adb_push_file` 函数优化：推送文件后自动执行 `chmod +x`
- 🔄 注册模块、测试报告确认页面及最终测试报告新增 `test_case_number` 展示列

### In Progress
- 🔄 优化MQTT主题监听输出方式：当前十六进制内容存 bin 文件、字符串内容存 txt 文件的模式存在解析兼容短板，已启动适配两种数据格式的统一改造方案研究

### Fixed
- 🛠️ 修复 `build.spec` 清单文件语法错误（缺少逗号导致模块加载失败）
- 🛠️ 修复 `push_and_prepare_tool` 中重复的 `chmod +x` 调用
- 🛠️ 修复记分牌测试验证逻辑：采用关键字符串匹配方案，解决特殊字符编码问题

### Security
- 🔒 源码保护：打包方案改为清单驱动，`.py` 文件不进入 dist（仅 `.pyc`）
